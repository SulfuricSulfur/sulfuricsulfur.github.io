Documentation

How the projectwas created; what technologies did you use? What did you write yourself and what did you get from somewhere else?

The project was created using one HTML file (the main index page), a javascript file (the file that holds the game logic) and a css document (a document which holds the presentation to change the look). The HTML page only contains the svg main, the body, and linking the javascript document and the css together to be displayed. The top of the javascript page holds basic info to make accessing info easier and quicker. It holds info about the brick (size, offset from the other bricks, the array of bricks, and the amount of bricks broken and total). There is also the score, the amount the score will increase by each time a brick is hit, and the number of lives. There is also the speed of the ball and the direction in which the ball is going. The top also contains the audio variables. The javascript has a number of functions in it. There is:

  • CreateBall
    • This function creates and sets a ball every time it is called.
  • RespawnBall
    • This checks to make sure that the ball the ball can move (if the ball was just spawned into the level or if the player just lost)
    • If the ball can’t move, put the ball in the middle of the paddle and set the direction to 0 until the ball is shot again.
  • CreatePaddle
    • Creates the paddle.
  • ShootBall
    • This checks to see if the ball can be shot. If it can, then set the moving direction to be negated (go to top of the screen) once the ball is shot.
  • CreateBrick
    • Creates the bricks. Passes in x and y position.
    • There is a function inside it to check to see if the ball has cullided with the brick. If it has, then remove the brick, add to the score, and add one to the amount of bricks that have been broken. Will also play audio if brick is hit.
  • DisplayHighScore
    • This gets localStorage and adds an item for the high score and the high score’s name.
    • Displays the high score and the name.
  • SetUpBricks
    • This makes a random number to be the number of bricks in a row and the number of bricks in a column. There will then be a for loop which loops through the rows and columns. The x and y for the bricks that will have an offset to the left and also the top.
  • ReserveSpd
    • Checks to see if the ball collides with the walls. If it does, bounce opposite way and play audio.
  • BallDie
    • Checks to see if the ball goes off the bottom of the screen. If it does, lose a life and restart the ball’s position and play audio.
  • updateText
    • Updates the text on the screen to display the score and the amount of lives.
  • Move
    • Checks to see if the ball can move.
    • If it can: multiple the speed by the direction for the x and y and then add it to the position for each.
  • BallPaddle
    • Checks to see if the ball’s bounding box has collided with the paddle’s bounding box. If it has, reverse the y’s direction. Also, check to see where the ball’s x position in relation to the paddle. Depending on where it is, it will affect the distance and where it goes.
  • movePaddle
    • Moves the paddle based on the mouse’s x position. The mouse should be within the bounds on the screen. If not, don’t move the paddle.
    • Gets half the width of the paddle and the current x position of the paddle. Subtracts half the width of the paddle from the mouse’s x to make the mouse in the middle.
  • Init
    • Loads all of the functions that create the brick, paddle, and ball.
    • Sets the number of bricks broken to 0.
  • Win
    • Checks to see if the amount of bricks broken is equal to the amount of bricks total.
    • If it is, play audio.
    • Tell player they have won.
    • Make sure the ball can’t move.
    • Set the ball to the middle of the paddle.
    • Reset the number of bricks.
    • Increase The score by a levelIncrease variable.
    • Increase the speed for the next level by 2.
    • Recreate new bricks for the next level.
  • Lose
    • Check to see if the lives have run out.
    • f it has: play lose audio.
    • Tell player they have lost and inform them of their score.
    • Instructs player to enter their name.
    • If the score is higher than the current highscore, than replace the leader board score and name with the recent one and upload it to the localStorage.
    • If there is no prior leader, then this will be the leader score.
    • Reset the game.
  • GameLoop
    • Run everything that requires to be constantly checked while the game is running.
    • The checking includes collision, if player has lost, if player has won, ball moving, updating score and lives, the paddle’s position, if the ball has collided with the bricks, and if the ball has died.
  • Make sure the init loads when window is opened.
  • Set the interval of the GameLoop to 16 so it can constantly run.

What new skills you had to learn to realize the project?

I have learned more about how to use javascript and svg in general. For a while, I had trouble with javascript and svg. It took me a while to get use to coding the elements, so it took a couple of weeks to get more comfortable with coding in the style. This includes spawning them in and their interactions with each other via collision. One particular collision method I learned about was using .getBoundingClientRect() in order to get a bound area rectangle around the element to check for collision. I was also able to implement multiple audio to the game that play depending on the situation (lose, win, or collision). I also learned more about the ball bouncing based on different collision from the tutorials on the brick breaker game via javascript and svg. I also have a localStorage Implemented in the assignment for high score. Onto of that, I also learned about taking input from a window box in order to collect the user’s name to display it along with the high score.
